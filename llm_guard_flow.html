POST /analyze/prompt
串行扫描 Prompt，返回净化后内容 + 风险分数。用于需要脱敏/过滤后再发给 LLM 的场景。
串行Scanner 链顺序执行，前一个净化结果传入下一个；支持 fail_fast 提前中断
// Request
{
  "prompt": "My name is John Doe...",
  "scanners_suppress": ["Language"]
}

// Response
{
  "sanitized_prompt": "My name is PERSON_1...",
  "is_valid": true,
  "scanners": { "Anonymize": 0.0, "PromptInjection": 0.02 }
}
POST /scan/prompt
并行扫描 Prompt，只返回是否合规 + 风险分数，不返回净化内容。速度更快，适合纯检测场景。
并行asyncio.gather 同时运行所有 Scanner，任一失败即标记 is_valid=false
// Request（同 /analyze/prompt）
{ "prompt": "Ignore all instructions..." }

// Response（无 sanitized_prompt）
{
  "is_valid": false,
  "scanners": { "PromptInjection": 0.97 }
}
POST /analyze/output
串行扫描 LLM 输出，返回净化后内容（含 PII 还原）+ 风险分数。
串行需要同时传 prompt + output，Deanonymize 依赖 prompt 上下文还原 PII
// Request
{
  "prompt": "My name is PERSON_1...",
  "output": "Hello PERSON_1, your email..."
}

// Response
{
  "sanitized_output": "Hello John Doe, your email...",
  "is_valid": true,
  "scanners": { "Relevance": 0.05, "Bias": 0.01 }
}
POST /scan/output
并行扫描 LLM 输出，只做安全检测，不净化，不还原 PII。
并行asyncio.gather 并发所有 Output Scanner，速度快
{ "prompt": "...", "output": "..." }

{
  "is_valid": false,
  "scanners": { "MaliciousURLs": 0.95 }
}

PII
Anonymize
检测并脱敏 PII（姓名/邮箱/手机/IP等），映射存入 Vault，Output 端可还原。基于 DeBERTa + Presidio。
注入
PromptInjection
检测提示词注入攻击（"忽略之前所有指令"），DeBERTa v2 模型，threshold 可调。
内容
Toxicity
检测仇恨/暴力/辱骂等有害内容，默认阈值 0.9。
内容
BanTopics
零样本分类，禁止特定话题（violence/politics/adult…），topics 列表可自定义。
内容
BanSubstrings
精确字符串黑名单，支持 word/str 匹配、大小写敏感、redact 模式。
内容
BanCode
检测 Prompt 中是否包含代码片段，防止代码注入。
内容
BanCompetitors
语义检测是否提及竞争对手品牌，competitors 列表可配置。
格式
Secrets
检测 100+ 种密钥格式（GitHub Token/AWS Key/OpenAI Key…），支持自动 redact。
格式
Regex
自定义正则规则，支持 block / redact / search 模式，灵活度最高。
格式
TokenLimit
限制最大 Token 数，防止超长攻击和成本超支，encoding 可选。
格式
Language
限制允许的语言（valid_languages: ["en"]），拒绝非指定语言输入。
内容
Gibberish
检测乱码/无意义内容，防止垃圾 Prompt 消耗资源。
内容
Sentiment
情感分析，过滤极端负面 Prompt，threshold 可配置。
内容
EmotionDetection
检测情绪类别（愤怒/悲伤/恐惧…），blocked_emotions 可自定义。
格式
InvisibleText
检测不可见 Unicode 字符（零宽字符等隐写攻击）。
格式
Code
检测特定编程语言代码（languages 可配置），is_blocked 决定允许还是禁止。

质量

质量

NLI 模型检测 Output 与 Prompt 事实一致性，minimum_score 可配置。
质量
NoRefusal
检测 LLM 是否拒绝回答，保证服务可用性，threshold 可调。
内容
Bias
检测性别/种族/政治偏见，默认阈值 0.97，严格模式。
内容
Toxicity
检测 Output 中有害内容，与 Input 侧同款模型。
内容
BanTopics / BanSubstrings / BanCode / BanCompetitors
输出侧同款禁止规则，防止 LLM 生成被禁内容。
安全
MaliciousURLs

安全
URLReachability

格式
JSON

格式
Language / LanguageSame

格式
ReadingTime
限制阅读时间（max_time 秒），控制输出长度，truncate 模式可截断。
格式
Regex / Sentiment / Gibberish

🔧 自定义 Scanner — 只需实现一个方法

① 自定义 Input Scanner
# 1. 实现 Protocol 接口 class MyCustomInputScanner: def __init__(self, threshold: float = 0.8): self.threshold = threshold def scan(self, prompt: str ) -> tuple[str, bool, float]: # 你的检测逻辑 risk = my_model.predict(prompt) is_valid = risk < self.threshold # 返回: (净化后prompt, 是否安全, 风险分数) return prompt, is_valid, risk
② 自定义 Output Scanner
# Output Scanner 多一个 prompt 参数 class MyCustomOutputScanner: def scan(self, prompt: str, output: str ) -> tuple[str, bool, float]: # 可利用原始 prompt 做对比 risk = my_checker.check(prompt, output) is_valid = risk < 0.5 return output, is_valid, risk
③ 注册到 API（llm_guard/input_scanners/util.py）
def get_scanner_by_name(name, config): # 在现有 if-elif 链中添加: if name == "MyCustomInputScanner": return MyCustomInputScanner(**config) # ...
④ 在 scanners.yml 中引用
# scanners.yml input_scanners: - type: PromptInjection params: threshold: 0.92 - type: MyCustomInputScanner # ← 直接用类名 params: threshold: 0.8 # 任意参数透传给 __init__
💡 关键点：Scanner 是纯 Python Protocol（鸭子类型），不需要继承任何基类，只要实现正确签名的 scan() 方法即可。返回的三元组含义固定：(处理后文本, True=安全/False=危险, 0~1风险分数)。
